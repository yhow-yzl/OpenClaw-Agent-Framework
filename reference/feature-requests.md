# Feature Requests - 功能需求

> 记录新功能需求和能力缺口，用于规划未来的改进方向
> 
> **位置说明：** 这不是 `.learnings/` 的核心文件，是扩展能力追踪

## 格式说明

```markdown
## [FEATURE-YYYYMMDD-XXX] 简短标题

**Type:** [feature/capability_gap/integration/enhancement]

**Summary:** 一句话描述这个需求

**User Need:**
使用者的具体需求或痛点

**Current Limitation:**
目前的限制或无法达成的原因

**Proposed Solution:**
建议的解决方案或实作方向

**Expected Benefit:**
实作后预期的好处

**Implementation Complexity:** Low / Medium / High

**Fields:**
- recurring_count: [需求被提及次数]
- priority: [low/medium/high/critical]
- status: [requested/planned/in_progress/completed]
- request_date: [需求提出日期]
```

## Type 说明

| Type | 说明 |
|------|------|
| **feature** | 全新的功能或服务 |
| **capability_gap** | 现有功能的能力不足 |
| **integration** | 与现有系统或服务的整合 |
| **enhancement** | 现有功能的增强或优化 |

## 优先级评估

| Priority | 说明 |
|----------|------|
| **Critical** | 阻碍核心功能运作 |
| **High** | 显著影响使用体验 |
| **Medium** | 有用但非必需 |
| **Low** | 好有更好，没有也可以 |

## 复杂度评估

| Complexity | 工作量 |
|------------|--------|
| **Low** | 1-3 天可完成 |
| **Medium** | 1-2 周可完成 |
| **High** | 需要 3 周以上 |
